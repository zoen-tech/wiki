# メンション運用ルール

メンション（`@`）は **「あなたに反応してほしい」というシグナル** です。
だからこそ、使い方と受け取り方の両方にルールが必要です。

## 📌 基本原則

### 1. メンションは「ボールを投げる」行為

メンションした時点で、相手にボールを渡したことになります。
**メンションした側は「相手が反応するまで待つ」状態**、メンションされた側は「ボールを返すまで自分のタスク」です。

### 2. メンションされたら必ず何かしらの反応を返す

返信が必要 → スレッド内で返信
返信不要 → 元メッセージに 👍 や 👀 などのリアクション

!!! warning "反応しないのは厳禁"
    無反応は「無視された」という印象を与え、相手は「気づいてないのか / 忙しいのか / 嫌がってるのか」と不安になります。
    **何かしらのアクションを返す** ことが信頼関係を保つ最低ライン。

### 3. 12時間以内に反応する

業務時間中なら **可能な限り早く**、遅くとも **12時間以内** にリアクション or 返信を返します。
（夜・早朝・休日にメンションされた場合は、業務時間内で対応すればOK）

12時間反応がない場合、サマリBotから「📩 返信漏れ」として通知が届きます。

## 🎯 メンションの使い分け

### `@個人名`（例：`@平川拓実`）

- **特定の1人** に明確に依頼・確認する場合
- 一番強いメンション。明確な対応を求めるとき

### `@all`

- **スペース全員** への一斉メンション
- 使用は最小限に。本当に全員が対応すべき内容のときだけ
- 例：障害連絡、緊急アナウンス、全員参加の会議連絡

!!! danger "@all の乱用は禁止"
    日常的な業務報告や、一部の人にしか関係ない話題で `@all` を使うと、**メンションの価値が下がります**。
    結果として「またかー」と無視される文化が生まれ、本当に重要なときに反応してもらえなくなります。

### メンションなし

- **誰か特定の人を名指しせず、見た人が拾えばOK** な共有・雑談
- スレッド名でスペース全体に「ふんわり共有」する場面で使用

## 📋 メンションされた側の対応フロー

```
@メンションされた
   ↓
内容を確認
   ↓
すぐ対応できる？
   ├─ Yes → スレッド内で返信して完了
   └─ No  → 👀 リアクションして「あとで対応する」と意思表示
              ↓
           5分以上かかる作業？
              ├─ Yes → Googleタスクに登録（期限設定必須）
              └─ No  → メモして当日中に対応
```

詳細は [タスク管理ルール](task-management.md) を参照。

## 💡 メンションする側のマナー

### ✅ 良いメンションの例

> @平川拓実
> 来週の定例の議題、以下で大丈夫ですか？
> - SEO順位レポート
> - 新規案件の進捗
> NGがあれば追加・差し替えお願いします。

→ 何を求めているかが明確、判断材料も提示。

### ❌ 悪いメンションの例

> @平川拓実 例の件どうしますか？

→ 「例の件」が不明、何を求めているかも不明。
   相手は「どの件？」「何を判断すればいい？」と確認の手間が発生する。

### 推奨：メンション時に含める3要素

1. **何の話か**（コンテキスト）
2. **何をしてほしいか**（依頼内容）
3. **いつまでに**（期限）

## ❓ よくある疑問

### Q. リアクションだけだと冷たい？

A. むしろ逆。**返信より素早く、相手のストレスも少ない** ので歓迎されます。
ZOEN-TECH では「了解しました」と書く代わりに 👍 リアクションを推奨しています。

### Q. メンションされたけど自分は関係なさそう…

A. 「関係なさそうだけど念のためメンションされた」場合は、👀 リアクションだけでもOK。
明らかに誤メンションなら、スレッド内で「これは○○さん宛じゃないですか？」と一言添えると親切です。

### Q. メンションされて即対応できないけど、忘れそう…

A. 👀 リアクションを付けつつ、**5分以上かかりそうなら即Googleタスクに追加** しましょう。
詳細は [タスク管理ルール](task-management.md) 参照。
